# OPENENDED — Creative Technology Specification
## 3D / Motion / Spatial Design System

> 设计目标：让用户感觉这是一个被精心设计的数字系统，而不是素材拼贴页面。

---

## 一、素材在页面中的角色定位

### 原则：3D 不是主角，是空间的呼吸

| 层级 | 角色 | 元素 | 存在感 |
|------|------|------|--------|
| L0 - 环境层 | 空间氛围 | 噪点纹理、微弱光晕 | 潜意识级 |
| L1 - 结构层 | 空间骨架 | 抽象几何线框、网格 | 低存在感 |
| L2 - 节奏层 | 滚动呼应 | 浮动几何体、视差元素 | 中等存在感 |
| L3 - 焦点层 | 品牌记忆点 | Hero 区抽象雕塑、Identity 区数字身份可视化 | 有意识但克制 |

### 每个区域的 3D 角色

1. **Hero 区** — 抽象几何雕塑（线框球体/多面体），缓慢旋转，响应鼠标位移
2. **产品区** — 无 3D，让产品本身说话。仅用微妙的视差深度
3. **内容区** — 浮动的抽象平面/线条，暗示"内容层叠"的空间感
4. **Identity 区** — 粒子系统构成的抽象人形轮廓，呼应"数字身份"概念
5. **Manifesto 区** — 极简，仅用光线/渐变营造纵深

## 二、素材如何被"再设计"

### 降噪流程

```
原始素材 → 去色/单色化 → 降低对比度 → 简化结构 → 
统一到配色系统 → 调整透明度(0.03-0.15) → 与页面节奏匹配
```

### 具体处理

1. **几何体** — 不用实体渲染，只用线框(wireframe)或极低透明度的面
2. **颜色** — 只使用品牌霓虹色的极低饱和度版本（opacity 3%-15%）
3. **动画** — 所有运动速度降到"几乎静止"（rotation: 0.001rad/frame）
4. **尺度** — 3D 元素尺度要大，但存在感要低（大而轻）

### 不使用外部 3D 模型文件
所有 3D 效果通过代码生成：
- CSS 3D transforms（几何线框）
- Canvas 2D（粒子系统、抽象线条）
- SVG 动画（几何图形变形）
- 纯 CSS 渐变 + blur（光晕效果）

**理由：** 零依赖、极致性能、完全可控、无版权风险

## 三、3D 与滚动/动线的协同方式

### 滚动叙事结构

```
[进入] 黑暗 → 几何体从虚无中浮现
  ↓ 滚动
[Hero] 线框球体缓慢旋转 + 鼠标视差
  ↓ 滚动
[Marquee] 2D 过渡带，重置空间感
  ↓ 滚动  
[产品] 3D 退场，产品占据全部注意力
  ↓ 滚动
[内容] 浮动平面暗示内容层叠
  ↓ 滚动
[Identity] 粒子聚散 — 数字身份的隐喻
  ↓ 滚动
[Manifesto] 回归极简，光线收束
  ↓ 滚动
[退出] 渐暗
```

### 视差层级

```
背景层 (z: -200) — 网格、大型光晕，滚动速度 0.3x
中间层 (z: 0)    — 主内容，滚动速度 1x  
前景层 (z: +60)  — 浮动几何碎片，滚动速度 1.5x
```

### 鼠标交互

- Hero 区几何体：鼠标移动 → 几何体微微倾斜（最大 ±5°）
- 全局：自定义光标跟随，hover 可点击元素时变大
- 产品卡片：hover 时微妙的 3D 倾斜（tilt effect, ±2°）

## 四、统一设计语言的保持策略

### 视觉一致性规则

1. **颜色约束** — 3D 元素只使用 4 个霓虹色 + 白色，且 opacity ≤ 15%
2. **线条粗细** — 所有线框统一 1px，与 border 系统一致
3. **圆角** — 无圆角，所有几何体保持锐利边缘（与品牌调性一致）
4. **字体** — 3D 场景中不出现文字（文字永远在 2D 层）
5. **间距** — 3D 元素遵循与 UI 相同的 8px 网格系统
6. **动画曲线** — 统一使用 cubic-bezier(0.16, 1, 0.3, 1)，与 UI 动效一致

### 素材融入检查清单

- [ ] 去掉素材后，页面依然完整可用？ → 必须是
- [ ] 素材能看出来源吗？ → 必须不能
- [ ] 素材在移动端表现如何？ → 必须优雅降级或隐藏
- [ ] 素材加载影响首屏时间吗？ → 必须不影响

## 五、性能优化与取舍策略

### 性能预算

| 指标 | 目标 |
|------|------|
| FCP (First Contentful Paint) | < 1.5s |
| LCP (Largest Contentful Paint) | < 2.5s |
| CLS (Cumulative Layout Shift) | < 0.1 |
| JS Bundle (3D 相关) | < 30KB gzipped |
| 帧率 | 稳定 60fps |

### 优化策略

1. **不用 Three.js** — 太重（~150KB）。用 CSS 3D + Canvas 2D 实现所有效果
2. **懒加载** — 3D 元素只在进入视口时激活
3. **requestAnimationFrame** — 所有动画用 rAF，不用 setInterval
4. **will-change** — 对动画元素预声明，触发 GPU 加速
5. **Intersection Observer** — 离开视口的动画自动暂停
6. **移动端降级** — 检测设备性能，低端设备关闭 3D，保留静态渐变
7. **Reduce Motion** — 尊重 `prefers-reduced-motion`，关闭所有动画

### 取舍原则

```
流畅度 > 视觉复杂度 > 交互丰富度

如果帧率 < 55fps → 降低粒子数量
如果帧率 < 45fps → 关闭 Canvas 动画，回退到 CSS
如果帧率 < 30fps → 关闭所有 3D，纯静态
```

## 六、实现方案

### 组件清单

1. **`<AbstractSphere />`** — Hero 区线框球体，CSS 3D transforms
2. **`<FloatingGeometry />`** — 浮动几何碎片，CSS 动画
3. **`<ParticleField />`** — Canvas 2D 粒子系统，用于 Identity 区
4. **`<AmbientGlow />`** — 多色渐变光晕，纯 CSS
5. **`<MouseParallax />`** — 鼠标视差容器，Framer Motion
6. **`<ScrollProgress />`** — 滚动进度指示器

### 技术栈

- CSS 3D Transforms — 几何线框
- Canvas 2D API — 粒子系统
- Framer Motion — 滚动驱动动画、视差
- CSS Gradients + Blur — 光晕效果
- Intersection Observer — 性能优化
